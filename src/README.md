# src/

The primary application code for the Gemini Live outbound-calling platform. This directory is organized as a collection of bounded contexts: each subdirectory owns one area of concern (Gemini integration, database, routing, etc.). Shared utilities live in `lib/` and `config/` to avoid duplication. The two entry files—`index.js` (process start) and `app.js` (Express factory)—compose all dependencies via explicit dependency injection, making the codebase testable and modular.

**Design philosophy:** No component imports another without explicit dependency injection. This prevents hidden coupling, circular requires, and makes it easy to mock dependencies during testing or swap implementations later.

## Directory map

| Directory | Purpose |
|-----------|---------|
| `config/` | Environment accessors, Gemini model registry, static constants (good-bye phrases, USD-INR rate, reconnection limits) |
| `lib/` | Shared utilities: call storage, outcome classification, phone formatting, pricing math, JSON safety helpers |
| `prompts/` | Gemini prompt templates, system instructions, runtime cues |
| `routes/` | HTTP endpoint handlers: Plivo webhooks, calls API, campaigns, brochures, employees, prompts, model config, analytics, health checks |
| `gemini/` | Gemini Live session management: session lifecycle, reconnection logic, message routing, tool execution, keepalive signals |
| `db/` | PostgreSQL entity CRUD + schema: campaigns, contacts, analyses, prompts, WhatsApp messages, employees |
| `dashboard/` | Dashboard server-side rendering: HTML layout, CSS styling, client JS bootstrap |
| `plivo/` | Plivo PSTN bridge and WebSocket audio streaming: outbound calls, hangup, mulaw↔PCM codec utilities |
| `jobs/` | Async jobs and cron workers: batch call engine, auto-callback scheduling, callback data sync |
| `integrations/` | Third-party API clients: WhatsApp (Evolution/Evo-Go), DeepSeek (post-call analysis via OpenRouter) |

## Entry points

- **`src/index.js`** is the process entry point (referenced in Dockerfile `CMD`, package.json `start`). It loads environment config, initializes the database, starts background jobs, creates the Express app, attaches the WebSocket server for Gemini audio bridging, and listens on the configured port.

- **`src/app.js`** is the Express application factory. It accepts a dependency object, mounts body parsers, serves the dashboard HTML, and attaches feature routers. You can import this directly if you ever need to run the HTTP server standalone (e.g., for testing) without the full index.js orchestration.

## Byte-identical dashboard

The dashboard HTML is generated by calling `getDashboardHtml()` from `dashboard/`, which concatenates layout HTML, inline CSS, and bootstrap JS into a single string. The output must be byte-identical across deployments to maintain consistent caching and verifiability. The SHA256 hash must match `2dee3b508bc3d0436cff2ef0d829c530a692a3e31df8f8b3903e1ccb30cd4460`.

Changes to dashboard layout, styles, or client-side JS should:
1. Update the rendered HTML (test locally with `npm start`, diff the served HTML)
2. Update the baseline snapshot in `docs/dashboard-baseline.html`
3. Recalculate and record the SHA256 hash
